1. Introduction: What is TTT?

TIECorps Tailoring Tool 3 is a program for creating Emperor's Hammer and TIE Corps uniforms.

Since the template files for the uniforms are 3D models, which not too many people can easily work with, this tool is for assembling the uniform parts from given personnel data (either entered via the GUI or imported from the TC database).

TTT3 generates a basic preview in POV-Ray (the Persistence of Vision raytracer), allowing the user to comfortably set camera position, lighting and some other options, and finally renders the final uniform image by calling POV-Ray to render the final image as per your selections.

While it would have been easier and faster to just render out template pictures and have TTT put them together, as the first version of the program did, there would have been some disadvantages.

First, when putting images together that look 3D, but are only 2D, the objects don't interact (don't reflect and don't cast shadows on each other). Elements that do not always appear on the exact same spot (like a 3rd class merit medal) would also look perspectively wrong when put to a location where they were not originally rendered at. You therefore end up with a kind of 2.5D look.

Second, working with the 3D data directly adds a new level of flexibility, like being able to set the camera and lighting, change the resolution, and so on.

A lot of work and time was put into creating these uniforms for the Emperor's Hammer Strike Fleet.

Any use of this material outside the EH (especially, but not restricted to, use for other SW clubs) require explicit permission from the Fleet Commander.

Some parts of the geometry were inspired and/or taken from freeware sources which prohibit commercial use (please see the list of contributors). Some parts of the design are legal property of Lucasfilm Ltd. Hence, the uniforms created by TTT3 can not be put to any commercial uses.

While TTT3 allows you to set the camera position freely, pictures which you want to upload to the TC database must clearly show your position (cuff of at least one arm) and rank (left breast).

While the POV scripts that come with TTT3 allow for a high degree of customization, pictures you want to upload to the TC database must (obviously) obey EH/TC uniform regulations.

Finally, you are using the program on your own risk. Flawless operation of TTT3 on your system can't be guaranteed.

3. Requirements

TTT3 uses the Persistence of Vision raytracer (POV-Ray) version 3.7 and beyond to create the uniform pictures. POV-Ray is freeware and can be downloaded from http://www.povray.org/. You can install it to any directory, TTT3 should be able to find it via the registry. If that fails, please open TTT3's configuration dialog and set the path to your POV-Ray installation manually.

TTT3 has been developed and tested primarily on Windows 10 64-bit. It has also been successfully tested on Windows 7 32-bit, and Windows 7 64-bit.

4. Rendering options

As with previous versions of TTT, most of the program functionality should be self-explaining. However, with the new 3D approach there are some aspects which are not that obvious. They will be explained in this section.

  • Cancel rendering: You can always cancel POV-Ray by pressing Alt-G.
  • Mosaic preview mode: There is a check box in the POV options (all previews) labeled "mosaic preview mode". With this enabled, POV-Ray will first generate a rough preview of the scene with large blocks, then refine them so more details become visible with every pass. Note that the final pass does not get done faster by this, so overall rendering time will increase.
  • Cloth detail (dress and duty uniforms only): There is a highly detailed cloth texture on the uniform that makes it look more realistic when viewed from close. But from far away, this leads to heavy aliasing that looks like a Moiré pattern (if you really want to see the effect, switch from auto to highest quality and raytrace at standard camera setting and resolution).

    To get a grip on that unpleasant affect, there are four versions of the cloth texture in different resolutions. With "Cloth detail" to "automatic", TTT3 will attempt to choose the best one based on two factors; output image width, and the distance between camera lens and uniform surface. Leaving that setting alone should work for most cases. However, the parameters for the texture choosing algo were set by a trial-and-error procedure since it is obviously not possible to test every configuration. So you may encounter situations in which you are not happy with the cloth detail. If you can't see it at all, increase the detail, if you get Moiré effects, decrease it. You may also want to save your scene settings and send them to me, for I might be able to improve the automatic algo to produce better results based on your experiment.

5. Tips from the author

Just a few things which I thought you might find useful.
  • Profiles: In TTT3 you can save profiles from the main windows as well as from each preview window. This might seem confusing at first, but serves a purpose, which is keeping the uniform data and the displaying of the uniform data separate. So, for example, you can import a new officer's profile, but display it with your favorite camera and light settings which you have stored in a separate config file.
  • Speed up: In some cases you may wish to only see a rough preview of a scene so you can judge how it would work out in the end. In such a case you wouldn't want to wait hours for the result. One of the biggest slowdowns in the raytracing process is the anti-aliasing. Switching it off makes the picture much rougher, but it will also go *a lot* faster. Getting rid of shadows and turning down the resolution are only points two and three on the list. Then there is the overall quality setting which goes from 9 down to 1. Here is what it does (from the POV-Ray help):
    9: Compute media and radiosity
    8: Compute reflected, refracted, and transmitted rays
    6, 7: Compute texture patterns, compute photons
    5: Render shadows, including extended lights
    4: Render shadows, but no extended lights
    2-3: Show specified diffuse and ambient light
    0-1: Just show quick colors. Use full ambient lighting only. Quick colors are used only at 5 or below
  • Image formats: TTT3 natively exports the generated image in PNG format.

6. Squadron patches

In order for a squadron patch to be displayed on the uniform, it must be in BMP, PNG or JPG (by exception) format, it must be in the data/squads directory and it must be named after the squadron (same spelling that it has in TTT3).

A size of 256 x 256 pixels is recommended, but not enforced. Just make sure it is square because otherwise it will appear distorted.

You can define transparent areas by alpha channel (PNG format only). As a fallback TTT3 will attempt to trace the outline of a non-transparent image. TTT3 will then auto-generate a b/w mask from it and store it in the same directory under originalname_mask.gif. You don't have to care about it, and if you ever modify an existing patch, TTT3 will generate a new one.

8. Known issues

The following bugs and/or problems of the TTT3 software are known, but not getting fixed anytime soon for some reason or another.

  • In some instances TTT3 may generate a 'Fatal error detected - Failed to execute script TTT3' warning. This has been resolved in some cases by relocating the 'TTT3' installation folder away from the Windows Desktop or Downloads/Pictures directories.
  • TTT3 may trigger a "Microsoft Defender SmartScreen" alert stating that it has prevented an unrecognized app from starting. Please be assured that TTT3 is safe to use. TTT3 has been submitted to Microsoft for analysis and deemed safe to use by the Windows Defender team. This warning is generated due to myself being a private / hobby developer who does not own the expensive licences that immediately earn trust from Microsoft. Because of this, trust for TTT3 has to be earned via its use, this process can take some time.
